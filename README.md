# 🎮 Feria Gamer 2026 — Videojuego Educativo sobre Ciberacoso

> **Proyecto Final — Programación Orientada a Objetos (2026-10)**
> Universidad del Norte · Barranquilla, Colombia
> V Feria Gamer — 28 de mayo de 2026

---

## 🎯 Descripción

Videojuego educativo 2D desarrollado en **Godot 4.6.1** que aborda la problemática
del bullying y ciberacoso de manera empática, no violenta y educativa.
El jugador toma decisiones, aprende estrategias de defensa positiva y desarrolla
empatía hacia las víctimas de acoso.

---

## 🛠️ Tecnologías

- **Motor:** Godot 4.6.1
- **Lenguaje:** GDScript (tipado estático)
- **Control de versiones:** Git + GitHub

---

## 🚀 Cómo ejecutar el proyecto

1. Descarga [Godot 4.6.1](https://godotengine.org/download) (versión estándar, no Mono)
2. Clona este repositorio:
   ```bash
   git clone https://github.com/krl0sconk/feria-gamer-game.git
   ```
3. Abre Godot → **Import** → selecciona la carpeta del proyecto
4. Presiona ▶️ para ejecutar

## 📁 Estructura del Proyecto

```
feria-gamer-game/
├── 📁 assets/           # Sprites, audio, fuentes, videos
│   ├── audio/           # Música y efectos de sonido
│   ├── fonts/           # Tipografías
│   └── images/          # Backgrounds, personajes, UI, sprites
├── 📁 scenes/           # Escenas Godot (.tscn)
│   ├── menus/           # Menú principal y pausa
│   ├── ui/              # HUD, diálogos
│   ├── characters/      # Player y NPCs
│   ├── levels/          # Niveles del juego
│   └── world/           # Elementos del mundo
├── 📁 scripts/          # Lógica GDScript (.gd)
│   ├── characters/      # Scripts de personajes
│   ├── managers/        # Gestores (GameManager, etc.)
│   ├── patterns/        # Implementaciones de patrones POO
│   ├── systems/         # Sistemas (diálogo, score, etc.)
│   ├── ui/              # Scripts de interfaz
│   └── utils/           # Utilidades y helpers
├── 📁 resources/        # Temas, shaders, materiales, datos
├── 📁 addons/           # Plugins de Godot
├── 📁 tests/            # Tests
├── 📁 docs/             # Documentación
│   ├── design/          # GDD, UML, wireframes
│   ├── technical/       # Arquitectura, decisiones técnicas
│   └── reports/         # Sprints, pruebas de usabilidad
└── 📁 ai/               # Contexto y herramientas de IA
    ├── context/         # Contexto del proyecto para IA
    ├── prompts/         # Prompts reutilizables
    ├── decisions/       # Decisiones de diseño
    └── conversations/   # Conversaciones relevantes con IA
```
